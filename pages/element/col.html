<col>

Description

The <col> tag defines one or more columns in a column group within a table and is used to apply styles to entire columns, instead of individual cells. It is used within a <colgroup> element that doesn't have a <span>.

Only a limited number of CSS properties effect it:

  • background: Will set the background for cells within the column. As the column background color is painted on top of the table and column groups, but behind background colours applied to the row groups, and the individual cells, backgrounds applied to table columns are only visible if every layer painted on top of them has a transparent background.
  • border: Only applies if border-collapse: collapse is set on the <table>.
  • visibility: The value collapse for a column results in all cells of that column not being rendered, and cells spanning into other columns being clipped. The space these columns would have occupied is removed. However, the size of other columns is still calculated as though the cells in the collapsed column(s) are present. Other values for visibility have no effect.
  • width: defines a minimum width for the column.

Attributes

The <col> element includes Global Attributes.

  • span: Specifies the number of columns a <col> element should span. Its default value is 1.
Summary

Content categories None.
Permitted content None; it is a void element.
Tag omission Must have a start tag and must not have an end tag.
Permitted parents A <colgroup> element that does not have a <span> attribute.
Implicit ARIA role No corresponding role
Permitted ARIA roles No role permitted
DOM interface HTMLTableColElement
